## 认知负荷评估设计（行为 + 主观，预留生理）

本系统的认知负荷评估设计，主要服务于「老龄认知障碍人群低认知负荷知识辅助」这一目标，为后续实验与商业计划书中的认知负荷部分提供技术落地支撑。

### 一、整体思路

- **多源信号**：当前阶段聚焦于「行为 + 主观」两类信号，未来预留与 EEG、眼动等生理信号的对齐接口。
- **统一时间轴**：所有事件与问卷结果均采用「时间戳 + 事件类型 + 关键参数」的结构化格式记录，便于后续与任意时序信号进行对齐分析。
- **低侵入与可选**：评估逻辑不影响正常使用流程，问卷为可选、简单、快速填写（3–5 题）。适合在演示/实验模式下开启，也可以按需要在生产环境中精简。

### 二、行为指标设计

行为数据主要在两类关键任务上采集：

- **患者教育阅读任务**：从输入主题、生成内容到分步浏览每个小节/步骤。
- **智能问答任务（Chat）**：从提问到拿到一次完整回答。

事件通过前端 `cognitiveLoad` 工具记录到本地（后续可接后端接口），统一格式为：

- `ts`：时间戳（毫秒）
- `event_type`：事件类型
- `task_id`：任务 ID（如 `pe_1678...`、`chat_1678...`）
- `session_id`：会话 ID（针对 Chat）
- `source`：来源（`patient_education` / `chat`）
- `params`：关键参数（JSON）

目前定义的事件类型包括：

- `task_start`：任务开始（记录主题/问题等）
- `task_end`：任务结束（记录耗时 `duration_ms`、视图模式等）
- `step_view`：分步模式下查看某一步（索引、总步数）
- `back`：回退操作（从哪一步回到哪一步）
- `click`：关键点击（如「next_step」「switch_mode」等）
- `error_or_repeat`：错误或重复操作（预留）
- `submit_questionnaire`：提交问卷（与主观结果关联）

这些行为指标可支持如下分析示例：

- 不同呈现模式（长文 / 分步 / 卡片）下的平均阅读耗时、步数。
- 不同任务类型（服药、复诊、日常照护）下的回退频率、下一步点击次数。
- Chat 回答长度与 `task_end.duration_ms` 之间的关系等。

### 三、主观问卷设计

主观认知负荷通过一个极简的 3–5 题小问卷采集，当前版本的问题示例：

1. 刚才这段内容理解起来难吗？（1=很容易，5=很难）
2. 需要反复看吗？（1=不需要，5=总是需要）
3. 信息量感觉如何？（1=太少，3=刚好，5=太多）
4. 整体阅读/理解负担如何？（1=很轻松，5=很吃力）

问卷在两类时机弹出：

- 患者教育：生成并浏览一段时间（例如 5 秒）后。
- Chat：一次问答完成后。

前端组件 `CognitiveLoadQuestionnaire` 在提交时调用 `recordQuestionnaire(taskId, sessionId, source, answers)`，写入：

- `ts`：提交时间戳
- `task_id` / `session_id` / `source`
- `answers`：`[{qid, value}, ...]`

同时自动记录一条 `submit_questionnaire` 行为事件，实现行为与主观结果的联结。

### 四、数据存储与导出

当前实现阶段，评估数据存储在前端浏览器的 `localStorage` 中：

- `cognitive_load_events`：行为事件列表
- `cognitive_load_questionnaires`：问卷记录列表

为便于实验和答辩演示，前端提供了一个管理页：

- 组件：`CognitiveLoadAssessmentView`（路由 `/cognitive-load`）
- 功能：
  - 行为事件与问卷记录的数量统计、任务数估计、平均问卷得分等。
  - 列表查看最近 N 条事件与问卷。
  - 一键导出为 CSV（含 UTF-8 BOM，便于 Excel 打开）。
  - 清空本地评估数据（实验重复初始化）。

后续如需在后端长期存储，可复用同一数据结构，将上述两个列表通过接口上报至后端数据库（如 MySQL 中新增评估表），依然保持「时间戳 + 事件类型 + 关键参数」的设计。

### 五、与 EEG / 眼动等生理信号的对齐预留

商业计划书中提到的未来方向包括 EEG、眼动等生理信号的认知负荷评估。本项目在当前阶段做了以下预留：

- 所有事件与问卷结果均带有精确时间戳，且不修改已经发生的历史记录，便于后处理对齐。
- 事件类型划分足够细（task_start / task_end / step_view / back / click），可以作为后期切割 EEG/眼动序列的锚点（如某步开始、某按钮点击等）。
- 问卷结果可以作为主观标签，与生理信号的复杂特征（频段功率、眼跳频率等）做关联分析。

在需要接入 EEG/眼动设备时，可以：

- 由外部采集软件记录独立的时间轴；
- 在试验任务开始时，系统触发特定事件（如 `task_start` + 唯一 trial_id），并在生理采集端同步记录；
- 事后通过 trial_id 与时间偏移，将两侧数据对齐到同一实验时间线。

### 六、与商业计划书的对应关系

- **商业计划书中的「低认知负荷」**：在本系统中落实为多形态知识装饰（长文/分步/卡片）、适老化主题与分步交互，以及行为 + 主观的认知负荷评估闭环。
- **「行为 + 主观 + 生理」三层评估结构**：当前已实现前两层，第三层通过时间戳与事件类型预留对齐接口。
- **「服药/复诊/日常照护」等高频场景**：既体现在知识内容（模板、患者教育主题）上，也体现在事件与问卷的参数设计中（task_id、source、keywords 可指向具体场景）。

以上内容可在答辩、路演或评审时作为「认知负荷评估模块」的技术说明，帮助评审理解本项目如何将认知负荷理论具体落在系统实现上，并如何支持后续的实验与扩展。
