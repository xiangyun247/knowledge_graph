# 智护银龄 —— 中期优化 TODO 清单

**目标**：在短期品牌与文案改造已完成的前提下，从**知识内容、知识装饰、认知负荷评估、适老化交互**等维度深化项目，使产品更贴近「老年认知障碍人群」与商业计划书中的「忆路康」定位。

**对齐商业计划书要点**：
- 知识装饰：同一知识多种呈现形式（长文 / 分步 / 卡片），降低外在认知负荷  
- 认知负荷量化：行为数据 + 主观量表，预留生理信号接口  
- 场景聚焦：服药提醒、复诊准备、日常照护、亲属联络、记忆相关  
- 适老化：简化语言、分步引导、大字号/高对比、少选项少步骤

| M2 | **示例知识数据** | 在 `scripts/init_database.py` 或单独 `data/` 下增加一份「老年认知障碍」示例知识（如：轻度认知障碍表现、日常照护要点、服药注意、防跌倒/走失），用于演示与测试；可与现有 init 数据并存。 | P1 |
| M3 | **患者教育主题预设** | 前端患者教育中心（或后端配置）中增加「认知照护」预设主题列表，如：轻度认知障碍老人居家注意事项、服药提醒要点、复诊前准备、家属如何与老人沟通。用户可一键选主题再微调。 | P1 |

## 二、知识装饰（同一内容多形态呈现）

| 序号 | 方向 | 具体内容 | 优先级 |
|------|------|----------|--------|
| M6 | **患者教育多版本输出** | 后端患者教育生成接口（或 Agent 输出）支持「呈现模式」参数：`long`（长文）、`step`（分步列表）、`card`（卡片式要点）。同一主题生成一份结构化内容后，按模式渲染为不同段落结构或前端展示形态。 | P0 |
| M7 | **分步版结构化规范** | 定义「分步版」输出格式：每步一句短句 + 可选配图说明；最多 5–7 步；每步字数上限（如 30 字）。便于前端做「一次只显示一步」的低认知负荷展示。 | P1 |
| M8 | **前端患者教育展示模式** | 患者教育结果页增加「切换呈现」：长文 / 分步 / 卡片。分步模式下一屏只显示当前步骤 + 下一步按钮；卡片模式用大字号、高对比度块状展示要点。 | P1 |
| M9 | **聊天回答的「简化版」选项** | 在 Chat 页或 Agent 层增加「简洁回答」或「分步回答」选项：同一问题可返回「详细版」与「三步以内简要版」，供不同认知负荷偏好使用。 | P2 |

## 三、认知负荷评估（行为 + 主观，预留生理）

| 序号 | 方向 | 具体内容 | 优先级 |
|------|------|----------|--------|
| M10 | **行为数据埋点** | 在前端关键路径埋点：任务/问题开始与结束时间、点击/翻页次数、回退次数、错误或重复操作。可先落在前端 state 或本地存储，再通过现有「历史记录」或新接口上报（不改变现有 API 契约时可先写前端，后端后接）。 | P0 |
| M11 | **认知负荷简易问卷** | 设计 3–5 道极简主观题（如「刚才这段内容理解起来难吗？」「需要反复看吗？」），在完成一次患者教育阅读或一次问答后弹出；结果与当次 task_id/session_id 关联存储，便于后续分析。 | P0 |
| M12 | **评估数据结构预留** | 行为与问卷数据采用「时间戳 + 事件类型 + 关键参数」形式记录；事件类型包含：task_start、task_end、step_view、back、submit_questionnaire 等，便于未来与 EEG/眼动等时序信号对齐。 | P1 |
| M13 | **管理端/实验端查看** | 增加简单管理后台或实验页面，增加「认知负荷评估」视图：按任务/用户汇总行为指标与问卷得分，支持导出。无后台时可先做 CSV 导出或只落库。 | P2 |

## 四、前端适老化与低认知负荷交互

| 序号 | 方向 | 具体内容 | 优先级 |
|------|------|----------|--------|
| M14 | **全局字号与对比度** | 提供「适老模式」或主题：整体字号放大一档、正文行距与对比度提高；关键按钮与链接更易辨认。可与现有主题切换并存。 | P1 |
| M15 | **关键页分步引导** | 患者教育、智能问答结果等「长内容」页，支持「分步展开」：默认只显示第一条/第一步，用户点击「下一步」再展开，减少单屏信息量。 | P1 |
| M16 | **入口与导航精简** | 首页/封面突出 2–3 个主入口（如：问一问、看提醒、患者教育），次要功能收进「更多」；避免一次展示过多入口导致「选择负担」。 | P2 |
| M17 | **语音播报（可选）** | 患者教育或问答结果支持「朗读」：调用 TTS 读当前段或当前步骤，方便视力或阅读能力较弱的用户。依赖现有 TTS 能力，可先做「朗读全文」再细化到「朗读当前步」。 | P2 |

## 五、Agent 与 RAG 的认知照护侧重点

| 序号 | 方向 | 具体内容 | 优先级 |

| M18 | **Agent 系统提示词强化** | 在 Agent 系统提示中明确：回答面向老年用户或照护者时，优先使用简短句、分条列举、避免长段落；对医学术语给出通俗解释或括号说明。 | P1 |

## 九、验收标准（中期）

- **知识装饰**：同一患者教育主题能以前端可切换的「长文/分步/卡片」至少两种形式展示。  
- **认知负荷**：完成一次阅读或问答后，可记录行为指标（如耗时、步数）并弹出简易问卷，数据可落库或导出。  
- **场景与内容**：有明确的「服药/复诊/日常照护」类任务或预设主题，且 RAG/Agent 回答更偏认知照护话术。  
- **适老化**：至少有一档「适老」展示选项（如字号/对比度或分步展开），并在文档中说明与商业计划书的对齐关系。
