# 患者教育：分步版与卡片版呈现规范（M7）

同一主题生成一份结构化内容（title、sections、summary）后，按「呈现模式」可渲染为三种形态。

## 分步版（step）

- **用途**：低认知负荷展示，一次只显示一步，适合老年或认知障碍人群。
- **数据来源**：由后端从 `sections` 自动派生，无需单独生成。
- **格式**：
  - `steps`: 数组，每项 `{ "text": "短句", "image_caption": "可选配图说明" }`
  - 最多 **5–7 步**（常量 `STEP_MAX_COUNT = 7`）
  - 每步字数上限 **30 字**（常量 `STEP_MAX_CHARS = 30`）
- **前端**：分步模式下仅展示当前步 +「下一步」按钮，可配合配图占位。

## 卡片版（card）

- **用途**：大字号、高对比度块状展示要点，便于快速扫读。
- **数据来源**：由后端从 `sections` 派生，每小节对应一张卡片。
- **格式**：`cards`: 数组，每项 `{ "title": "小节标题", "content": "正文" }`

## 长文版（long）

- 默认形态：按 `sections` 顺序展示完整小节与 summary，与现有患者教育页一致。

## 后端返回

患者教育生成接口返回的 `patient_education` 对象统一包含：

- `title`, `sections`, `summary`, `share_text`（原有）
- `steps`：分步版数据（已派生）
- `cards`：卡片版数据（已派生）

前端根据用户选择的「呈现模式」切换展示即可，无需再次请求。
