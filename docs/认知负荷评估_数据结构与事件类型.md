# 认知负荷评估：数据结构与事件类型（M12）

行为与问卷数据采用「时间戳 + 事件类型 + 关键参数」形式记录，便于与 EEG/眼动等时序信号对齐。

## 存储位置

- 前端：`localStorage` 键 `cognitive_load_events`（行为事件）、`cognitive_load_questionnaires`（问卷结果）。
- 后端：可后续增加接口上报，格式与下文一致。

## 行为事件格式

每条事件为一条 JSON 对象：

```json
{
  "ts": 1709123456789,
  "event_type": "task_start",
  "task_id": "pe_1709123456789",
  "session_id": "chat_abc123",
  "source": "patient_education",
  "params": {}
}
```

### 事件类型（event_type）

| 类型 | 含义 | 典型 params |
|------|------|-------------|
| `task_start` | 任务/问题开始 | task_id, source, (topic/question) |
| `task_end` | 任务/问题结束 | task_id, duration_ms, (view_mode) |
| `step_view` | 分步模式下查看某一步 | task_id, step_index, total_steps |
| `back` | 回退（如上一步） | task_id, from_step, to_step |
| `click` | 关键点击（如翻页、切换模式） | task_id, target (e.g. next_step, switch_mode) |
| `error_or_repeat` | 错误或重复操作 | task_id, action |
| `submit_questionnaire` | 提交认知负荷问卷 | task_id, session_id, answers |

### source

- `patient_education`：患者教育阅读任务
- `chat`：聊天问答任务

## 问卷结果格式

```json
{
  "ts": 1709123500000,
  "task_id": "pe_1709123456789",
  "session_id": "chat_abc123",
  "source": "patient_education",
  "answers": [
    { "qid": "q1", "value": 2 },
    { "qid": "q2", "value": 1 }
  ]
}
```

问卷题目 id（qid）与题目定义见前端「认知负荷简易问卷」组件。
