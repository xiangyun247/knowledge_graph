# 演示脚本（5–8 分钟）

适用于比赛答辩、项目展示等场景，按固定流程演示系统核心功能。

---

## 一、演示前准备

- 确保前后端已启动（`http://localhost` 或 `http://localhost:8080` 可访问前端，`http://localhost:5001/docs` 可访问 API）
- 预先注册一个演示账号（如 `demo / Demo123456`），或使用已有测试数据
- 若有示例图谱或知识库，可提前构建，演示时直接展示

---

## 二、演示流程（约 5–8 分钟）

### 1. 系统介绍（约 30 秒）

**话术示例**：

> 本系统是「胰明」医学知识图谱生成与智能问答系统，支持从 PDF、文本等数据源自动构建知识图谱，并基于 LangGraph Agent 进行智能问答。系统采用 Neo4j 图数据库、MySQL、Chroma 向量库，支持多轮对话、患者教育生成、知识库管理等功能。

**操作**：打开首页，简要展示界面布局（登录入口、导航栏等）。

---

### 2. 登录 / 角色（约 30 秒）

**操作**：

1. 点击「登录」，输入演示账号（如 `demo / Demo123456`）
2. 登录成功后，展示用户信息或角色标识（若有 doctor/patient/admin 等角色）
3. 说明：系统支持多角色，不同角色可访问不同功能模块

---

### 3. 知识图谱构建与可视化（约 1–2 分钟）

**操作**：

1. 进入「数据上传」或「知识图谱」页面
2. 上传一个示例文件（如 `data/examples/sample_pancreatitis.txt`）或选择已有图谱
3. 触发构建（若为异步，可展示任务进度）
4. 构建完成后，进入图谱可视化页面，展示节点、边、缩放、拖拽等交互
5. 简要说明：图谱由 LLM 提取实体与关系，写入 Neo4j 与 MySQL，支持多图谱切换

---

### 4. 智能问答（约 1–2 分钟）

**操作**：

1. 进入「智能问答」或「Chat」页面
2. 输入问题，如「胰腺炎有什么症状？」
3. 展示流式输出或完整回答
4. 若有来源展示（来自图谱 / 来自文献），指出右侧的来源卡片
5. 可选：进行多轮追问，如「它怎么治疗？」，展示 session 记忆

**话术**：Agent 会按需调用图检索、文献检索等 RAG 工具，综合生成答案。

---

### 5. 患者教育（约 1 分钟）

**操作**：

1. 在问答界面选择「患者教育」意图，或进入患者教育专门页面
2. 输入主题，如「急性胰腺炎患者注意事项」
3. 展示生成的结构化内容（标题、分节、温馨提示）
4. 若有配图，可简要展示

**话术**：系统可将医学知识转化为患者易懂的说明，便于医患沟通。

---

### 6. 知识库管理（约 30 秒–1 分钟）

**操作**：

1. 进入「知识库」或相关入口
2. 展示知识库列表、文档上传与检索
3. 输入检索关键词，展示向量检索结果
4. 说明：文档经分块与嵌入后存入 Chroma，供 Agent 文献检索使用

---

### 7. 多模态（语音 / OCR，如有）（约 30 秒）

**操作**：

1. 在 Chat 页面点击麦克风按钮，录制语音
2. 展示语音转文本结果并填入输入框
3. 若有图片解读：上传医学影像或报告图片，展示 OCR + 解读结果

**话术**：系统支持语音输入与图文结合问答，便于临床场景使用。

---

## 三、时间分配建议

| 环节       | 建议时长 | 可压缩场景     |
|------------|----------|----------------|
| 系统介绍   | 30 秒    | 直接进入登录   |
| 登录 / 角色| 30 秒    | 已登录可跳过   |
| 图谱构建与可视化 | 1–2 分钟 | 用已有图谱演示 |
| 智能问答   | 1–2 分钟 | 核心环节，建议保留 |
| 患者教育   | 1 分钟   | 可合并到问答   |
| 知识库管理 | 30 秒–1 分钟 | 简要带过     |
| 多模态     | 30 秒    | 环境不支持可跳过 |
| **合计**   | **5–8 分钟** |              |

---

## 四、常见问题与应对

| 情况           | 应对                         |
|----------------|------------------------------|
| 问答超时       | 提前说明 2G 云主机可能较慢，本地演示更流畅 |
| 图谱无数据     | 使用 init 脚本中的示例图谱，或提前构建   |
| 知识库为空     | 跳过知识库环节，或提前 ingest 少量文档   |
| 语音/OCR 不可用 | 说明为可选功能，跳过多模态环节       |

---

## 五、演示结束语

**话术示例**：

> 以上是系统的核心功能演示。系统支持从数据到图谱、从问答到患者教育的完整流程，并具备多模态与知识库扩展能力。感谢观看，欢迎提问。
