## 二、后端数据库问题

### 2.1 MySQL游标兼容性问题（高优先级）

**问题描述**：`cursor.fetchone()` 在不同MySQL客户端版本返回类型不同（tuple vs dict），导致 `repository.py` 数据持久化失败。

**影响范围**：评估记录无法保存到数据库。

**修改方案**：

```python
# repository.py - get_mysql_client 返回值处理
def get_mysql_client():
    client = MySQLClient()
    # 强制使用 dict cursor
    client.cursor = client.connection.cursor(dictionary=True)
    return client
```

或在 `save_assessment` 方法中使用列名索引：

```python
# 替换 cursor.fetchone() 调用
result = cursor.fetchone()
if result:
    # 原代码：assessment_id = result[0]
    # 修复后：dict cursor 直接用列名
    assessment_id = result['id'] if isinstance(result, dict) else result[0]
```

### 2.2 EEG特征表缺失（中优先级）

**问题描述**：当前只有 `behavior_events` 和 `nasa_tlx_answers` 表，EEG数据表预留字段存在但无独立存储。

**修改方案**：

```python
# repository.py 新增方法
def save_eeg_features(self, assessment_id: int, eeg_features: Dict) -> bool:
    """保存EEG特征数据（预留表）"""
    try:
        self.mysql.cursor.execute("""
        CREATE TABLE IF NOT EXISTS cognitive_eeg_features (
            id BIGINT AUTO_INCREMENT PRIMARY KEY,
            assessment_id BIGINT NOT NULL,
            alpha_power DECIMAL(10,6),
            beta_power DECIMAL(10,6),
            theta_power DECIMAL(10,6),
            theta_beta_ratio DECIMAL(8,4),
            signal_quality DECIMAL(3,2),
            recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            INDEX idx_assessment (assessment_id)
        ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
        """)
        
        self.mysql.cursor.execute("""
        INSERT INTO cognitive_eeg_features 
        (assessment_id, alpha_power, beta_power, theta_power, theta_beta_ratio, signal_quality)
        VALUES (%s, %s, %s, %s, %s, %s)
        """, (
            assessment_id,
            eeg_features.get('alpha_power'),
            eeg_features.get('beta_power'),
            eeg_features.get('theta_power'),
            eeg_features.get('theta_beta_ratio'),
            eeg_features.get('signal_quality')
        ))
        self.mysql.connection.commit()
        return True
    except Exception as e:
        logger.error(f"保存EEG特征失败: {e}")
        return False
```

```python
def test_cognitive_assessment_e2e():
```

### 5.1 认知负荷阈值个性化（中优先级）

**问题描述**：当前阈值固定（0.33/0.66），无用户个性化调整机制。

**修改方案**：

```python
# fusion_orchestrator.py 新增用户基线方法
def set_user_baseline(self, user_id: str, baseline_score: float) -> None:
    """设置用户认知负荷基线（用于个性化阈值）"""
    self._user_baselines[user_id] = baseline_score

def _compute_personalized_threshold(self, user_baseline: float) -> tuple:
    """基于用户基线计算个性化阈值"""
    # 低负荷：基线的70%以下
    # 高负荷：基线的130%以上
    low_threshold = user_baseline * 0.7
    high_threshold = user_baseline * 1.3
    return (low_threshold, high_threshold)
```

### 6.1 Redis缓存认知中间状态（低优先级）

**问题描述**：认知评估中间状态未利用Redis缓存。

**修改方案**：

```python
# cognitive/repository.py 新增
def cache_assessment_session(self, session_id: str, partial_data: Dict, ttl: int = 3600) -> bool:
    """缓存评估会话中间状态"""
    try:
        import redis
        r = redis.Redis(host='localhost', port=6379, db=1)
        r.setex(f"cognitive_session:{session_id}", ttl, json.dumps(partial_data))
        return True
    except Exception:
        return False

def get_cached_session(self, session_id: str) -> Optional[Dict]:
    """获取缓存的评估会话"""
    try:
        import redis
        r = redis.Redis(host='localhost', port=6379, db=1)
        data = r.get(f"cognitive_session:{session_id}")
        return json.loads(data) if data else None
    except Exception:
        return None
```

```javascript
import api from '@/api'
api.cognitiveLoad.assessCognitiveLoad()
```

### 9.4 API重复引入混乱（低优先级）

**问题描述**：各API模块的 `request` 引入方式不统一：
- 有的 `import request from '@/utils/request'`
- 有的 `import { request } from '../utils'`

**修改方案**：统一为一种引入方式（推荐后者，统一从 `../utils` 引入）：
```javascript
// 统一使用
import { request } from '../utils'
```

### 9.5 后端app.py过大（高优先级）

**问题描述**：`backend/app.py` 达到 **3000行**，单文件过大导致：
- 启动速度慢
- 路由难以维护
- 热更新效率低

**当前路由分布**（估计）：
- Agent路由
- 知识图谱路由
- 上传/构建路由
- 历史记录路由
- 搜索路由
- 认知负荷路由
- 患者教育路由
- 多模态路由（STT/OCR/TTS）

**修改方案**：

```
backend/
├── app.py                      (精简：只保留中间件、跨域、CORS配置，<200行)
├── routers/                    (新建：路由目录)
│   ├── __init__.py
│   ├── agent.py               (从 agent_router.py 拆分)
│   ├── kg.py                  (知识图谱路由)
│   ├── upload.py              (上传/构建路由)
│   ├── history.py             (历史记录路由)
│   ├── search.py              (搜索路由)
│   ├── cognitive.py           (认知负荷路由)
│   ├── patient_education.py   (患者教育路由)
│   └── multimodal.py          (多模态路由)
└── services/                   (可选：业务逻辑抽离)
    ├── __init__.py
    ├── agent_service.py
    └── kg_service.py
```

注册方式：
```python
# app.py 精简后
from fastapi import FastAPI
from routers import agent, kg, upload, cognitive, patient_education, multimodal

app = FastAPI()

app.include_router(agent.router, prefix="/api/agent", tags=["Agent"])
app.include_router(kg.router, prefix="/api/kg", tags=["知识图谱"])
app.include_router(cognitive.router, prefix="/api/cognitive-load", tags=["认知负荷"])
# ...
```

### 9.6 工具函数重复（中优先级）

**问题描述**：`backend/` 下存在多个功能相似的文件：
- `tasks.py` (403行) vs `celery_tasks.py` (297行) - 任务执行逻辑重复
- `hadoop_service.py` (658行) vs `hadoop_api.py` (461行) - Hadoop操作重复

**问题文件清单**：

| 文件对 | 重叠功能 | 建议 |
|-------|---------|------|
| `tasks.py` vs `celery_tasks.py` | 知识图谱批量构建 | 合并到 `celery_tasks.py` |
| `hadoop_service.py` vs `hadoop_api.py` | HDFS操作、MapReduce | 合并服务层，API只做路由 |
| `query_router.py` vs `query_service.py` | 问答路由 | 合并，`query_router.py` 只做路由 |

**修改方案**：

```python
# backend/
# 合并后的结构
tasks.py                    # 只做任务入口和类型定义
celery_tasks.py             # 包含所有Celery任务实现（合并后的）
hadoop_service.py           # Hadoop服务（合并后的，只保留服务类）
hadoop_api.py               # 删除，路由移到 routers/hadoop.py
query_service.py            # 问答服务（合并后的）
query_router.py             # 只做路由
```

### 9.8 重构风险控制建议

1. **每次重构只改一个文件**：提交前确保功能不受影响
2. **保留旧文件做备份**：删除前先注释掉，等新文件稳定后再删
3. **编写回归测试**：确保重构后API行为不变
4. **分阶段提交**：
   - 第一阶段：拆分组件（不改变功能）
   - 第二阶段：提取复用逻辑
   - 第三阶段：删除重复代码
